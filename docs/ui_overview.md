# User Interface Overview

The XAUTOMATA user interface provides tools to configure, monitor, and analyze complex systems through a combination of data management and visual dashboards.

The interface is organized into three main functional areas:

- [Data Manager](data_manager_overview.md)
- [Dashboards](dashboards_overview.md)
- [Administration](administration_overview.md)

These areas allow users to configure the system model, collect and organize operational data, and analyze it through visual dashboards.

---

## Data Manager

The **Data Manager** is the configuration area of the platform.

Here users define the entities that represent the monitored environment, such as customers, infrastructure objects, metrics, and services.

These entities form the internal model used by XAUTOMATA to represent real-world systems.

Examples of entities managed in this section include:

- Customers and Sites
- Contacts
- Infrastructure Objects
- Services
- Metrics and Metric Types
- Monitoring Probes

The Data Manager acts as the foundation of the platform: dashboards and analytics rely on the data defined in this area.

---

## Dashboards

Dashboards provide visual access to operational data.

Each dashboard is composed of multiple **widgets**, which display metrics, statistics, and aggregated information.

Dashboards allow users to:

- monitor system status
- analyze trends
- investigate anomalies
- explore infrastructure data

Different dashboards focus on different domains, such as:

- IT infrastructure monitoring
- network monitoring
- cloud cost governance

---

## Widgets

Widgets are the building blocks of dashboards.

Each widget retrieves data from the platform and presents it using visual components such as charts, tables, or grids.

Widgets are organized into **widget groups**, which represent functional domains.

Examples include:

- Infrastructure monitoring widgets
- Network monitoring widgets
- Cloud cost analysis widgets
- Service management widgets

---

## Administration

The **Administration** section allows administrators to configure platform-level settings.

This includes:

- user management
- system probes
- notification systems
- access control configuration
- dashboard and widget management

Some advanced configuration features are available only to **Super Admin** users.

---

## Navigation Structure

The platform navigation menu reflects the logical structure of the system.

Major sections include:

- Customers and infrastructure repositories
- Operational tracking tools
- System administration
- Analytical dashboards

This organization allows users to move from configuration tasks to operational monitoring in a structured workflow.
