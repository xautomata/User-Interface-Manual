# Navigation

The XAUTOMATA user interface is organized into functional areas that support both system configuration and operational analysis.

The main navigation menu allows users to move across repositories, tracking tools, dashboards, and administrative sections.

## Main Navigation Areas

The interface is structured around the following main sections:

- Customers
- Tracking
- Administration
- Dashboards

Each section contains specific tools and views dedicated to a particular area of the platform.

## Customers

The **Customers** area contains the main repositories used to model organizations and monitored environments.

It includes:

- **Client Repository**
  - Customers
  - Sites
  - Contacts

- **Objects Repository**
  - Groups
  - Objects
  - Metric Types
  - Metrics
  - Services

This section is mainly used to configure the entities that represent the customer environment and its monitored infrastructure.

## Tracking

The **Tracking** area provides tools to manage operational events and planned activities.

It includes:

- Calendars
- Downtimes
- Dispatchers

These tools are used to coordinate maintenance windows, service interruptions, and operational workflows.

## Administration

The **Administration** area contains system-level configuration pages.

It includes:

- Users
- Virtual Domains
- Probes
- Probe Types
- Messages
- Notification Providers
- Notification Provider Types
- Cloud Cost Registration
- Analytical Accounting
- Super Admin tools

This area is typically used by administrators to configure access, integrations, and advanced platform settings.

## Dashboards

Dashboards provide the analytical and operational views of the platform.

They allow users to monitor infrastructure, network, service, and cloud cost data through configurable widgets.

The dashboards available to a user depend on permissions and customer configuration.

## Navigation Logic

In general, users move through the interface according to the following pattern:

1. configure entities and relationships
2. define operational or analytical structures
3. access dashboards and widgets for monitoring and analysis

This makes the navigation structure consistent with the overall workflow of the platform.
